🔄 Flujo de Datos

Cómo viajan los datos desde que el usuario guarda una bitácora hasta que aparece una insignia.

Guardar bitácora — Flujo completo

sequenceDiagram actor U as Usuario participant B as Bitacora.jsx participant LS as localStorage participant SL as saveLog.php participant SA as send_alert.php participant DB as MySQL participant EMAIL as Email Admin U->>B: Completa formulario y guarda B->>LS: Guarda bitácora (offline-first) B->>SL: POST /api/saveLog.php SL->>DB: INSERT INTO bitacoras DB-->>SL: OK SL-->>B: {"success": true} alt mood=frustrado/cansado ó ayuda=true B->>SA: POST /api/send_alert.php SA->>EMAIL: PHPMailer → Admin EMAIL-->>SA: Enviado SA-->>B: {"success": true} end B->>B: Actualiza UI + dispara cálculo de insignias

Sincronización al entrar al dashboard

sequenceDiagram actor U as Usuario participant DL as DashboardLayout participant LS as localStorage participant FL as fetchLogs.php participant DB as MySQL U->>DL: Navega al Dashboard DL->>LS: Lee practicante_token alt No hay token DL->>U: Redirige a /login else Token válido DL->>FL: GET /api/fetchLogs.php FL->>DB: SELECT bitacoras + fellows + badges DB-->>FL: Datos FL-->>DL: JSON con bitacoras + fellows DL->>LS: Actualiza practicantes_bitacoras + practicantes_fellows DL->>DL: dispatchEvent('storage') → re-render components DL->>DL: calculateUserBadges() → CelebrationOverlay si hay nuevas end
